02 Model Overview and Purpose

2.1 Model Purpose

The model is designed to score retail credit applicants on their probability of default within the loan term. Scores are used for binary approve/decline decisions at loan origination. The model is not approved for use in pricing, capital allocation, or IFRS 9 expected credit loss calculations without a recalibration layer (see Finding PERF-02).

2.2 Algorithm

The model uses XGBoost (eXtreme Gradient Boosting), an ensemble of decision trees trained with gradient descent. XGBoost was selected because it handles non-linear interactions between credit features, manages missing values natively, and is widely validated in academic literature for credit scoring tasks. The model produces a probability score in [0, 1], where higher values indicate higher predicted default risk.

03 Data Quality and Governance

3.1 Dataset Description

The German Credit Dataset (UCI Machine Learning Repository / OpenML ID: credit-g) contains 1,000 loan applicants from a German bank with 20 features covering account status, loan characteristics, personal attributes, and collateral. The dataset is a standard benchmark for credit scoring research and has been used in published academic work since 1994.

04 Model Methodology Review

The validation team reviewed the model's algorithm choice, feature set, and training configuration for theoretical soundness and compliance with internal model standards.

4.1 Algorithm Suitability

XGBoost is appropriate for this use case. It captures non-linear relationships between features that logistic regression would miss. The model was trained with 200 estimators, max depth 4, and learning rate 0.05 — settings consistent with industry practice for credit scoring.

4.2 Overfitting Assessment

Finding PERF-03: Training AUC = {{ "%.4f"|format(train_auc) }} vs Test AUC = {{ "%.4f"|format(test_auc) }}. The gap of {{ "%.4f"|format(overfit_gap) }} indicates the model has over-learned training patterns and may not generalise well to new populations. Recommendation: Apply stronger regularisation and consider feature reduction.

4.3 Feature Set Review

All 20 features are customer-reported or bank-derived attributes standard in retail credit origination (account status, loan purpose, savings, employment, demographics). No features were identified as potential proxies for protected characteristics beyond existing governance controls.

5.2 Calibration

Calibration measures whether the model's predicted probabilities are consistent with observed default rates. A well-calibrated model is required for IFRS 9 and regulatory capital applications.

Finding PERF-02 (RED): Maximum calibration gap = {{ "%.4f"|format(calibration_gap) }}. Brier Score = {{ "%.4f"|format(brier_score) }}. The model's probabilities are significantly miscalibrated and must not be used as direct probability estimates without a calibration post-processor (Platt scaling or isotonic regression).

06 Sensitivity and Stress Testing

6.1 Decision Boundary Stability

Finding ADV-01 (RED): Boundary instability rate = {{ "%.1f"|format(instability_rate * 100) }}%. All borderline applicants (predicted score within ±5% of the threshold) flipped their approval/rejection decision when a single feature was perturbed by 5%. This represents a material fairness and consistency risk.

Recommendation: Implement score banding with a buffer zone of ±0.05 around the decision threshold. Applicants within this band should be referred for manual review rather than receiving an automated decision.

9.3 Overall Assessment

The model demonstrates acceptable discriminatory power for a credit scoring application (OOT AUC = {{ "%.4f"|format(oot_auc) }}, KS = {{ "%.4f"|format(oot_ks) }}). Feature importance is stable, economically interpretable, and consistent across the validation period. However, material weaknesses in calibration and boundary stability limit the model's applicability to rank-ordering decisions only.

The overall risk rating is {{ overall_rating }}. Conditional approval is granted for use in loan origination rank-ordering, subject to the remediation of RED findings and implementation of the monitoring plan in Section 8.
